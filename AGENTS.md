# EstiMat — правила для ИИ-агентов

Полные правила проекта — в `CLAUDE.md` (архитектура, стек, git, миграции, безопасность).
Ниже — обязательный минимум для любого агента, делающего коммиты.

## Версионирование и changelog (обязательно для каждого коммита)

- **Каждый коммит повышает версию приложения на 0.01** (0.25 → 0.26 → … → 0.99 → 1.00).
  Текущая версия — верхняя запись массива `CHANGELOG` в `client/src/changelog.ts`.
- Перед коммитом добавить запись **в начало** `CHANGELOG`: `version` (предыдущая + 0.01),
  `date` (сегодняшняя, YYYY-MM-DD), `changes` — краткие формулировки, понятные
  пользователям-сметчикам: что изменилось в интерфейсе/поведении, без технических
  деталей (имён файлов, роутов, слов «рефакторинг», «миграция»).
- Те же пункты продублировать в теле коммита блоком:

  ```
  Изменения:
  - <пункт 1>
  - <пункт 2>
  ```

- Чисто технические коммиты без видимых пользователю изменений тоже повышают версию —
  с одной строкой вида «Технические улучшения и исправления».
- `APP_VERSION` (пользовательская версия из changelog) не заменяет `BUILD_ID`
  (техническая метка сборки для баннера обновления) — механизмы независимы.
